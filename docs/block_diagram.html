Popis bloků

Programovací prostředí

Cílová platforma je Nordic nRF5340 (dual-core Cortex-M33, BLE 5.0). Firmware je postaven na Zephyr RTOS přes nRF Connect SDK 2.x. Zephyr poskytuje preemptivní vlákna, k_timer, semafory, fronty zpráv (k_msgq) a perzistentní flash úložiště (NVS). Ladění přes RTT (bez sériového portu). Systémový tick = 10 000 Hz pro pokrytí 1 kHz haptic smyčky.

Motorové řízení – low-level

Jádro tvoří SimpleFOC přepsaný do C (bldc_motor.c, bldc_driver_3pwm.c). Používá voltage-mode FOC se SVPWM / Sine PWM a driverem DRV8311H v 3-PWM režimu. Enkodér TMAG5170 (14-bit magnetický, Zephyr sensor driver, SPI) dodává úhel rotoru. Startovací sekvence najde elektrický nulový bod a výsledek uloží do NVS flash. Center-aligned PWM synchronizuje ADC trigger na střed periody pro eliminaci switching noise. haptic_loop() běží @1 kHz přes k_timer → semafor → dedikované vlákno (prio 0). Veškerá matematika akcelerována HW FPU + CMSIS DSP (arm_math.h).

Haptika – high-level

Čtyři módy: SMOOTH (plynulá odporová síla, IIR α=0,03 filtruje rychlost mezi tikmi) a DETENT 16/12/8 kroků (diskrétní zarážky, spring-type, IIR α=0,30 zachovává snapping). Hermite smoothstep (0,5°–2,5°) vyhlazuje dead-zónu – zabraňuje buzzingu ze šumu enkodéru LSB. Virtual click detekuje přitlačení osy (gesture zone) a hold timer (500 ms = Enter, 2 s = long-hold) – aktuátor plně nahrazuje fyzické tlačítko. Výstupem jsou callbacky step_cb a virtual_click_cb.

Uživatelské rozhraní

remote_control.c – hierarchické stavové menu (FSM přes menu_level_t): Root → Control / Temperature / Filament / Calibration / MMU / Macros / Printing / Z-offset. Edit screen (param_desc_t) s granularitou kroku dle rychlosti otáčení. Výstup na SSD1309 OLED (ui_display.c, framebuffer). Aktualizace displeje se neprovedou přímo z haptic callbacků (race condition vůči render smyčce) – místo toho atomic pending flag + push z main loop. Po 5 s idle se zobrazí status screen (XYZ, teploty, fan, progress).

BLE NUS komunikace

BLE 5.0 peripheral s Nordic UART Service (NUS) (Zephyr BT stack). Protokol: cmd:<id>:<payload> – 16-bit sekvenční číslo, gateway odpovídá ack:<id>:ok / err:<reason>. Dedikované TX vlákno (k_msgq, 16 zpráv, prio 7) – oddělen od haptic smyčky, BLE latence neovlivní motorové řízení. Příchozí stav tiskárny parsován do ble_printer_state chráněné k_mutex.

Gateway – Raspberry Pi 3

Python service na RPi 3. gateway.py – bleak (asyncio BLE klient) pro připojení k nRF5340. protocol.py (ProtocolTranslator) – NUS payload → G-code nebo přímý passthrough. moonraker.py (MoonrakerClient) – G-code přes Moonraker HTTP API do Klipperu; periodické čtení stavu tiskárny (poloha, teploty, progress) a odeslání zpět na zařízení přes NUS notify.
